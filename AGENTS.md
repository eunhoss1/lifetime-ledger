# AGENTS.md

## 프로젝트 규칙
- Vite, React, TypeScript, Dexie, Tailwind CSS, vite-plugin-pwa로 로컬 우선 개인 가계부 PWA를 만든다.
- 의존성은 최소로 유지한다. 실제 위험이나 복잡도를 줄이지 않는 패키지는 추가하지 않는다.
- MVP에서는 서버 동기화, 로그인, Supabase, Firebase를 구현하지 않는다. 나중에 붙일 수 있도록 repository 경계만 유지한다.
- 금액은 정수 KRW로 저장한다. 날짜는 `YYYY-MM-DD`, 월은 `YYYY-MM` 형식으로 저장한다.
- 도메인 로직은 React 컴포넌트 밖에 둔다. 계산, 반복지출 규칙, 월마감 규칙, 백업 검증은 테스트 가능한 도메인 모듈에 둔다.
- IndexedDB 접근은 repository 모듈을 통해서만 한다. UI 코드에서 Dexie 테이블을 직접 호출하지 않는다.
- 사용자 데이터는 특별한 이유가 없는 한 soft delete로 처리한다.
- JSON 백업/복구는 핵심 기능으로 취급한다. 스키마 변경 시 백업 검증과 테스트도 함께 갱신한다.
- 닫힌 월은 다시 열기 전까지 읽기 전용이다.
- 수정 전에는 기존 파일을 먼저 확인하고 사용자 변경사항을 보존한다. 명시적 승인 없이 파괴적인 git/파일시스템 명령을 실행하지 않는다.

## 테스트 규칙
- 반복지출, 월마감, 백업/복구, 날짜/금액 계산에는 Vitest 테스트를 추가하거나 갱신한다.
- 주요 UI 흐름이 생기면 Playwright 테스트를 추가한다.
- 완료 보고 전에는 타입체크와 관련 테스트를 실행한다.
- 테스트를 실행하지 못하면 정확한 이유를 보고한다.

## UI 규칙
- 모바일 우선, 빠른 입력 우선으로 만든다. 첫 화면은 마케팅 페이지가 아니라 실제 가계부 화면이어야 한다.
- 명확한 폼, 표, 필터, 작은 요약 카드를 선호한다.
- 매일 반복 사용을 느리게 만드는 장식적 UI는 피한다.
- iPhone 폭과 데스크톱 폭 모두에서 텍스트가 읽히고 겹치지 않게 한다.
- 사용자에게 보이는 문구와 프로젝트 문서는 기본적으로 한국어로 작성한다. 제품명이나 코드 식별자처럼 필요한 경우에만 영어를 유지한다.

## 데이터 안전
- 복구 과정에서 IndexedDB를 조용히 덮어쓰지 않는다. 검증, 요약, 확인 단계를 둔다.
- 전체 백업은 UTF-8 JSON으로, 거래 이동성은 CSV로 내보낸다.
- 앱 데이터는 IndexedDB에만 둔다. service worker는 앱 shell/assets만 캐시하고 금융 기록은 캐시하지 않는다.
- 작업 프로젝트는 `C:\dev\lifetime-ledger` 같은 일반 개발 폴더에 둔다. 백업은 OneDrive 동기화가 아니라 private GitHub 저장소와 JSON 내보내기를 기준으로 한다.
